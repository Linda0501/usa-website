How Mammoth 101:

Installation:

  1. Locate the github project made by mwilliamson, the title of the project is python-mammoth
    1. https://github.com/mwilliamson/python-mammoth
  2. Clone the repository in a convenient location on your computer
  3. To install mammoth run the following command
    1. “pip install mammoth”

Requirements and Recommendations for documents to process properly:

  1. Mammoth can process the following word document procedures correctly:
    1. Headings
    2. Lists
    3. *Tables
    4. Footnotes and endnotes
    5. Images
    6. Bold, italics, underlines, strikethrough, superscripts and subscripts
    7. Links
    8. Link breaks
  2. We find that mammoth has difficulties processing and formatting tables, to avoid incorrect formatting and issues, it is recommended that when possible, a screenshot of tables be taken and used rather than the table itself.
  3. ALL images must be in a PNG File format, X-EMF Files will not display properly
  4. Mammoth is a great tool to convert and is far better than any other convertor we’ve found, however it isn’t perfect and requires quite a bit of dumbing down of a document
  5. Once the proper changes have been made to the document format, place it into a folder, and make sure that within the folder there is another folder that is titled output-dir

  1. In this example, the document desired to be processed is Steiner Tree, and when mammoth processes it, the output will be put into output-dir along with all of the images used in the mammoth document

Converting Word Documents to HTML Files

  1. In order to properly use images inline the following command must be used to convert the document
    1. “mammoth document.docx --output-dir=output-dir”
    2. This will convert the document to html, and place it in the output-dir along with all images used inside of document.docx
  2. Mammoth allows other ways to modify documents, and the steps to convert documents with
  3. additional features can be found in the READ.me of the mammoth github page

Uploading the HTML Document to the website

  1. The SUSA website requires a template above the HTML Document in order to correctly show the document.
  2. Furthermore add the endblock to the bottom of the document which is:
    1. {% endblock %]
  3. The header can be added to the Mammothed document in 1 of 2 ways
    1. Manually copy and pasting the template to the top of the Mammothed document is one option
    2. To automatically enter the header, use the Pandoc universal document converter which can merge documents
    3. To download Pandocs, run the installer found on https://pandoc.org/installing.html
    4. Copy the template and paste it into output-dir
    5. To merge the documents run the command: pandoc -s template.html mammothed.html -o output.html
    6. Unfortunately, this will output a merged document, however output.html will automatically put a header, make sure to delete the area before the template, and also the paragraph symbols around the template
    7. Also please delete the </html> at the bottom of the document before you add the endblock above
  4. Now that the template has been added, it is time to upload it to the desire location within the USA-Website github repository. Make sure that the outputted HTML document, does not leave the folder with all of its images, because it will then not display the images correctly
  5. The following link explains how to properly upload the output to the website
    1. https://docs.google.com/document/d/1ZsrAKC9Z8TcuDiSBLnk2PbE9JvXhSX9C2_m-dirWabY/edit
