Purpose

This post in particular will help guide you through Front End vs Backend and hopefully clear up some misconceptions people may have regarding to how they interact and any overlap between the two. We will describe the general structure of frontend vs backend and use the common example of a LAMP stack to briefly illustrate how the back end functions. To fully understand the differences in function between the front end and the backend and get an introduction to web architecture, please visit this blog. Because we already have tutorials on front end development, I will instead provide my opinion on the order in which to learn the languages presented in the front end section. We will also be exploring a few backend languages (Django, Node, PHP) and go over a brief overview of the pros and cons of each one.

A brief note: for the remainder of this post, we will assume 'front end' is two words, and 'backend' is one. There seems to be a lot of variations of the phrase 'front end' on the internet (e.g. front-end, frontend), but for the sake of standardization I will just pick this version and stick with it.

What is a Front End

To start off, we'll talk about the part of a website you're probably most comfortable with. The Front End of a website is everything the user sees in their browser. Typically we use a mixture of HTML, CSS, and some sort of Javascript to create everything on the front end. I should also note here that technically design prototyping plays a huge role in front-end development so you should also have a working knowledge of UX (user experience) principles and Gimp/Adobe Suite (to quickly prototype page layouts) but we're going to skip over that as it's not my forte.

To work on the front end of a site, you should start off by becoming proficient in HTML. HTML tends to be fairly basic with the most important distinction being class vs id. If you want a refresher on that subject, it is covered in our CSS tutorial (Part 2). Everything else is fairly basic and as long as you have a general idea of what you want to code, it shouldn't be too hard to look up the individual tags you need. For a basic overview, please visit our tutorial

After you learn HTML, you need to learn CSS to make it look nice. For a brief refresher on what CSS is and what it does, see Part 2 and Part 3. Since there are so many different CSS properties and layouts (e.g. Flexbox, Grid), I would recommend building a few pages and learning the most basic properties first. After that, you should focus on positioning which is the most challenging and important thing you can do with CSS (Learn bootstrap to make this a lot easier). If you're feeling comfortable with that, move on to CSS animations.

The last thing you need to know when working on a front end is javascript. Javascript is responsible for most of the interactivity and moving parts you see on websites. Because teaching javascript is such a large task and comprable to teaching Python or some other standard programming language (because it is one), we will not even be attempting to do so in our tutorials. Instead, we will provide some areas to focus on. The important take aways are listed below.

  • Start with Vanillajs. A lot of people learn more complex libraries such as JQuery as a crutch, and will write hard to read or bad code. Learning libraries is not necessarily bad, but save it for after you are comfortable with javascript as a language unless it is absolutely necessary.
  • Javascript can be object oriented too! Javascript, like other languages such as Python or Java can be object oriented albeit a bit quirky. Learn how to use callbacks and higher order functions.
  • As of ES6, use 'let' and 'const' to create variables, NOT 'var'. ES6 (also ECMAScript or Javascript 6) is currently the industry standard for writing javascript. Using let will create variables as you would normally expect var to do in other languages such as python, while creating a variable with var inside a function will instead define the variable globally, which may lead to issues.
  • Speaking of ES6, LEARN ES6! A list of changes can be found here. The most important and confusing of these changes is probably the introduction of 'arrow functions' which is just a shorthand method of writing anonymous functions.
  • Understand how to change HTML/CSS with js. You can actually also build entire front ends with just javascript. However your main usage of this will probably involve changing some CSS for an element after a button click, adding an animation to a web page, or modifying a web page based on form information.

Once you have mastered all of the above, you can start learning about different js libraries. Libraries we have used in the past here at SUSA include JQuery for DOM manipulation, leaflet.js for mapping data, and d3.js for data visualization (though I do not recommend jumping straight into d3). You can also learn some javascript frameworks like React.js (I will introduce frameworks later). The final note I would like to leave you off on for this section is that there are multiple ways of doing things in front end development whether it is adding an action to a button click or positioning images or other elements. Familiarize yourself with all of these methods just in case you have to work on a project where other people have different ways of doing these tasks.

What is a Backend?

Generally speaking, the backend is everything that makes a website run that you can't see. It generally consists of an operating system, a web server, a database, and a server side programming language. You may have heard of a LAMP stack, which stands for:

  • Linux - The Operating System
  • Apache - The Webserver
  • Mysql - The Database
  • PHP - The Server Side Language

In SUSA's case, the OS (Linux - Debian) and webserver (nginx) are handled by the OCF so we only need to worry about the database and server side language. While we don't currently use a database (or have any plans for one), our server side lanugage is Python. Different server side languages have their own strengths. For example, python is very good at using machine learning on realtime data, though our predecessors here at webdev may have chosen it due to their familiarity with Python through their CS classes. Node.js however, is very good with real-time two way connections between the client and server.

So how does a backend work? typically when you enter a url into your browser, it pings a webserver which is hosting a website. From there, it gives the backend instructions on which web pages to render and provides it with any information stored in a database that it would need. For a more in-depth explanation of how exactly the different parts of the stack work together please read this.

So What is Django? (Frameworks)

We've gone over the basics of a backend and each individual part, but if you've ever talked to Anthony or me about our website, you may be asking yourself, "Why haven't you mentioned Django yet?" For context, we in webdev like to refer to our backend as a Django backend. This is a bit misleading in that Django is actually a backend Python web framework. We don't typically make this distinction because no one in SUSA really cares besides us and most people who do care can reasonably infer what we mean. In spite of our laziness, if you ever get into web development, it is important to understand what a framework is.

A Framework is a package built upon an existing language (e.g. Python) and makes development easier by either making it easier to organize your code or providing tools and shortcuts/solutions for dealing with common development problems. For example, the main role of a backend is to handle http requests and return pages. Now you can code this from scratch every time you build a web application, but Django provides built in methods to serve web pages in a fraction of the time. Backend frameworks also help with using cookies, handling POST and GET requests, and easily organizing your code.

PHP

PHP as a language has been around for a very long time and for good reason. Created by Rasmus Lerdorf in 1994, PHP was not originally inteded to be a programming language. It was just a script to bind tools together to build a Personal Home Page. But because of a passionate group of developers and its open source nature, it grew in scale and purpose, becoming a very widely used language. Because it was originally hacked together, PHP has a history of being inconsistent with its conventions and having a lot of security flaws which have contributed to its poor reputation throughout the years.

However, at the time of this writing (PHP 7.3) PHP is a very fast backend language that also has a very large and established community. Recent additions to the language offer support for anonymous functions and the ability to specify return type declarations. Despite being labled as notoriously hard to learn, the latest versions share surprising similarities in syntax to C and shouldn't be too hard to pick up. The main thing to watch out for when learning PHP is that many of the tutorials you'll find on the web are very outdated. PHP used to suffer from inefficiency and many things from previous versions would now be considered bad practice. The pros and cons of PHP in general are as follows:

Pros
  • Very simple to start using
  • Lots of database languages are compatible
  • PHP developers are always in high demand as it's a very widely used language
  • Wordpress is built on PHP
Cons
  • Lots of outdated tutorials
  • Scalability requires extensive planning
  • Very hard to fully master
  • Used to be very bad as a language in general (2012)

I want to clarify something regarding the last bullet point for the pros of PHP. Wordpress is a very popular website building tool for non-developers. While it uses PHP for its backend, it also uses PHP to select front end components from its database and plug them directly into the front end. This is because PHP can generate html when used inside a .php file that contains html. Likewise, html can also pass information back to PHP via the traditional POST and GET requests.

Admittedly, I don't have too much experience with frameworks outside of Python and Node, but I do have some experience using Laravel.
